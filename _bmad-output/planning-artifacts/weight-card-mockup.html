Design Philosophy: Visual Hierarchy & Progress Feedback

  • ‚úì Four-Layer Feedback System: Ring segment (macro distribution) + Colored text (identity) + Progress bar (quantity) + Emoji (quality)
  • ‚úì Progress Bar Placement: Below macro label, above value - creates visual flow from identity ‚Üí progress ‚Üí current value
  • ‚úì Subtle Border: 0.5px border around progress bar provides definition without visual noise
  • ‚úì Weight Trend Logic: Only show green ‚Üì when current_weight < rolling_average (celebrate wins, hide setbacks)
  • ‚úì "Coach, Not Judge" Philosophy: Positive reinforcement emphasized over punishment

Weight Card Specifications

  • Layout: Horizontal card with 3 sections: Current Weight (left) | Rolling Avg (center-right) | Trend (right)
  • Current Weight: 28pt bold, neutral dark (#2D2D2D), displayed in kg
  • Rolling Average: 14pt semibold, gray (#666) - Day 1: single weight, Day 2: avg of 2 days, ... Day 7+: rolling 7-day average
  • Average Label: Shows "Avg" (not "7-Day Avg") to remain accurate from Day 1
  • Trend Display (conditional): Green arrow ‚Üì (24pt) + amount in kg (e.g., "-0.4", 11pt semibold, green #4CAF50)
  • Trend Logic: Show green ‚Üì ONLY if current_weight < rolling_average, hide completely otherwise
  • Tap Behavior: Opens full weight history screen (Phase 2: line graph, all entries)

Macro Display Specifications

  • Format: current/goal (e.g., "128/160g")
  • Protein Goal: weight_kg √ó 2 (e.g., 80kg user ‚Üí 160g protein)
  • Fats Goal: weight_kg √ó 1.0 (e.g., 80kg user ‚Üí 80g fats, minimum for hormone health)
  • Carbs Goal: Flexible - fills remaining calories after protein + fats are allocated
  • Auto-Recalculation: Macro goals update automatically when user logs new weight (protein/fats scale with current bodyweight)
  • Calculation Example (80kg user, 1900 cal diet phase):
    • Protein: 160g √ó 4 cal/g = 640 cal
    • Fats: 80g √ó 9 cal/g = 720 cal
    • Carbs: (1900 - 640 - 720) √∑ 4 = 135g (540 cal)
  • Smiley Logic (% of goal):
    • üòä Green: ‚â•70% of goal (e.g., 112g+ for 160g protein target)
    • üòê Yellow: 50-69% of goal (e.g., 80-111g)
    • üòû Red: <50% of goal (e.g., <80g)
  • Progress Bar (positioned below label, above value):
    • Width: 40px, Height: 3pt, Border-radius: 2px
    • Border: 0.5px solid rgba(0,0,0,0.1) for subtle definition
    • Background: Light gray (#E0E0E0, 30% opacity)
    • Fill color: Matches macro color (Protein=Green #4CAF50, Carbs=Orange #FFA726, Fats=Red #EF5350)
    • Fill percentage: current √∑ goal √ó 100% (e.g., 128/160 = 80% filled)
    • Animation: 300ms ease-out on page load
    • Placement: Emoji ‚Üí Label ‚Üí Progress Bar ‚Üí Value (top to bottom)
    • Purpose: Visual reinforcement of progress (complements emoji feedback)
