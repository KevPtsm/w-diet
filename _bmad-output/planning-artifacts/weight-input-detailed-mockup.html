Comparison: Input Methods

Method A: Simple Text Field

  • PRO: Fastest input (type and done)
  • PRO: Familiar pattern (standard iOS)
  • PRO: Minimal UI (just field + keyboard)
  • PRO: Works great for daily quick logging
  • CON: Requires keyboard (blocks view)
  • CON: Typos possible (61.9 vs 69.1)

Method B: iOS Picker Wheel

  • PRO: Native iOS pattern (familiar)
  • PRO: No keyboard needed
  • PRO: Prevents invalid inputs
  • PRO: Precise scrolling control
  • CON: Slower than typing
  • CON: Requires more vertical space
  • CON: Harder to jump to values far from current

Method C: +/- Stepper Buttons

  • PRO: One-handed friendly
  • PRO: Large tap targets (accessibility)
  • PRO: Haptic feedback on each tap
  • PRO: Fine control (+0.1) and quick jumps (¬±1.0)
  • CON: Slowest for large changes
  • CON: More UI complexity

Recommendation: Method A (Simple Text Field)

Why Method A wins for w-diet POC:

Speed matters: Users weigh themselves daily in the morning routine. They want to enter weight in 3 seconds and move on. Typing "69.1" on decimal pad ‚Üí Tap Save is the fastest path.

Familiar pattern: Every nutrition app (MyFitnessPal, Yazio, Lose It) uses text field input for weight. Users expect it.

Minimal POC scope: Method A requires ~20 lines of SwiftUI code. Method B (picker) requires custom wheel component. Method C (stepper) requires increment/decrement logic + haptics + state management.

Phase 2 consideration: Can add picker or stepper as alternative input method in Settings ("Prefer picker wheel for weight entry"). Start simple, iterate based on user feedback.
